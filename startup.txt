================================================================================
  MCKO — Инструкция по запуску и использованию
================================================================================


БЫСТРЫЙ СТАРТ
-------------

1. Клонировать репозиторий и перейти в папку:

     git clone <url> mcko
     cd mcko

2. Запустить скрипт установки:

     bash run.sh

   При первом запуске скрипт:
   - установит python3 и pip3 (если отсутствуют)
   - установит xclip (на Linux)
   - установит Python-зависимости из requirements.txt
   - создаст файл .env из шаблона и остановится с просьбой заполнить его

3. Открыть .env в любом текстовом редакторе и указать API-ключ:

     OPENROUTER_API_KEY=sk-or-xxxxxxxxxxxxxxxxxxxxxxxx

   Остальные параметры можно оставить по умолчанию.

4. Запустить повторно:

     bash run.sh

   Приложение запустится в фоне. Никакого окна не появится — это нормально.


ОСТАНОВКА
---------

Приложение работает через watchdog-процесс. Чтобы полностью остановить:

     kill <PID>

  PID выводится в терминале при запуске run.sh.
  Также можно найти процесс вручную:

     pgrep -f watchdog.sh


ХОТКЕИ
------

  Глобальные (работают в любом приложении):

    Ctrl + Space          — переключить окно чата (показать / скрыть)
    Ctrl + Shift + Space  — показать окно чата (только открыть, не закрывать)
    Ctrl + Alt + Space    — обработать содержимое буфера обмена (фоновый режим)

  В окне чата:

    Enter               — отправить сообщение
    Shift + Enter       — перенос строки (новая строка без отправки)
    Ctrl + V            — вставить текст или изображение из буфера обмена
    Escape или × кнопка — скрыть окно (история сохраняется)
    Ctrl + C / X / Z    — стандартные операции с текстом


СЦЕНАРИЙ 1 — ОКНО ЧАТА (Ctrl + Space)
--------------------------------------

Используется для диалога с AI в течение сессии.

1. Нажмите Ctrl + Space — появится окно чата.
2. Введите вопрос или задачу в поле внизу.
3. Нажмите Enter — сообщение отправится, ответ появится по мере генерации.
4. Продолжайте диалог — история переписки сохраняется до закрытия приложения.
5. Нажмите Escape, кнопку × в заголовке окна, или Ctrl + Space повторно —
   окно скроется, история останется.

Окно не отображается в панели задач и трее ни в каком состоянии.
Единственный способ вернуть его — хоткей (Ctrl+Space или Ctrl+Shift+Space).
Закрытие терминала, через который запускалось приложение, не останавливает MCKO.

Вставка изображения:
  - Скопируйте изображение в буфер обмена (PrintScreen, Ctrl+C в просмотрщике и т.д.)
  - В поле ввода нажмите Ctrl + V
  - Вместо самого изображения появится метка: [added pic1]
  - Метка удаляется целиком: достаточно нажать Backspace рядом с ней
  - Можно вставить несколько изображений — они пронумеруются: pic1, pic2, ...
  - Нажмите Enter — изображение будет отправлено вместе с текстом


СЦЕНАРИЙ 2 — ФОНОВЫЙ ЗАПРОС (Ctrl + Alt + Space)
--------------------------------------------------

Используется без открытия окна. Работает в любой момент.

Для текста:
  1. Выделите нужный текст в любом приложении и скопируйте (Ctrl + C).
  2. Нажмите Ctrl + Alt + Space.
  3. Подождите несколько секунд.
  4. Вставьте результат (Ctrl + V) — в буфере будет ответ AI.

Для изображения:
  1. Скопируйте изображение в буфер (PrintScreen, Ctrl+C и т.д.).
  2. Нажмите Ctrl + Alt + Space.
  3. Подождите несколько секунд.
  4. Вставьте результат (Ctrl + V) — в буфере будет текстовый ответ AI.

Никакое окно не открывается. Процесс полностью фоновый.


КОНФИГУРАЦИЯ (.env)
-------------------

  OPENROUTER_API_KEY    API-ключ (получить на https://openrouter.ai/keys)
  OPENROUTER_MODEL      Модель, например: openai/gpt-4o, google/gemini-pro
  SYSTEM_PROMPT_1       Системный промпт для окна чата
  SYSTEM_PROMPT_2       Системный промпт для фонового запроса
  HOTKEY_WINDOW         Переключить окно (по умолчанию: <ctrl>+<space>)
  HOTKEY_SHOW           Только показать окно (по умолчанию: <ctrl>+<shift>+<space>)
  HOTKEY_CLIPBOARD      Хоткей буфера (по умолчанию: <ctrl>+<alt>+<space>)

  Формат хоткеев (pynput): <ctrl>, <alt>, <shift>, <space>, <f1>...<f12>, a...z

После изменения .env перезапустите приложение (kill + bash run.sh).


ЛОГИ
----

Все события пишутся в файл mcko.log рядом с main.py:

     tail -f mcko.log

Там отображаются: запросы к API, срабатывания хоткеев, ошибки, перезапуски watchdog.


WATCHDOG И АВТОПЕРЕЗАПУСК
--------------------------

Приложение запускается через watchdog.sh. Если main.py аварийно завершится,
watchdog автоматически перезапустит его через 3 секунды. Каждый перезапуск
фиксируется в mcko.log с временной меткой.


СТРУКТУРА ФАЙЛОВ
----------------

  run.sh          — установка зависимостей и запуск
  watchdog.sh     — цикл автоперезапуска
  main.py         — основной процесс
  .env            — конфигурация (заполняется вручную)
  .env.example    — шаблон конфигурации
  mcko.log        — журнал событий (создаётся автоматически)
  app/            — логика: API, хоткеи, буфер обмена, история
  ui/             — интерфейс: окно, область чата, поле ввода
