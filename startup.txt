================================================================================
  MCKO - STARTUP AND USAGE
================================================================================

QUICK START
-----------

1. Clone and enter the repository:

     git clone <url> mcko
     cd mcko

2. Start installation/run script:

     bash run.sh

   The script installs dependencies from local `packages/`, creates `.env` from
   `.env.example` if needed, and starts the app through `watchdog.sh`.

3. Fill `.env` if it was created from the template:

     OPENROUTER_API_KEY=<base64-or-raw-openrouter-key>

4. Restart:

     bash kill.sh
     bash run.sh

The app starts hidden. That is expected.


STOPPING
--------

Use:

     bash kill.sh

or manually stop watchdog/main processes if needed.


HOTKEYS
-------

Global hotkeys:

  Ctrl + Space          toggle chat window
  Ctrl + Shift + Space  show chat window
  Ctrl + Alt + Space    solve current clipboard content in background
  Ctrl + Shift + S      take screenshot and insert it into the input field

Chat window:

  Enter                 send
  Shift + Enter         newline
  Ctrl + V              paste text or image
  Escape / ×            hide window


MAIN GUI FLOW
-------------

1. Press Ctrl+Space.
2. Type a task or paste one/more images.
3. Press Enter.
4. The response worker calls `GeometryPhotoSolver`.
5. The UI shows only the final answer.

Image paste:

  - pasted images become labels like `[added pic1]`
  - several images are allowed
  - image labels are converted to OpenRouter `image_url` content blocks on send


BACKGROUND CLIPBOARD FLOW
-------------------------

Text:

  1. Copy text.
  2. Press Ctrl+Alt+Space.
  3. Wait.
  4. Paste the result from clipboard.

Image:

  1. Copy screenshot/image.
  2. Press Ctrl+Alt+Space.
  3. Wait.
  4. Paste answer-only text from clipboard.


CONFIGURATION
-------------

Required:

  OPENROUTER_API_KEY

Models:

  PHOTO_SOLVER_KIMI_MODEL   primary solver, default moonshotai/kimi-k2.6
  PHOTO_SOLVER_QWEN_MODEL   parser/OCR/visual extraction, default qwen/qwen2.5-vl-72b-instruct
  PHOTO_SOLVER_LLAMA_MODEL  verifier/repair, default meta-llama/llama-4-maverick

Hotkeys:

  HOTKEY_WINDOW
  HOTKEY_SHOW
  HOTKEY_CLIPBOARD
  HOTKEY_SCREENSHOT

`OPENROUTER_API_KEY` may be raw or base64-encoded. `app/config.py` decodes
base64 values that do not start with `sk-`.

Legacy variables are not used:

  OPENROUTER_MODEL
  OPENROUTER_MODELS
  SYSTEM_PROMPT_1
  SYSTEM_PROMPT_2


CURRENT SOLVER ROUTES
---------------------

Text-only:

  Kimi K2.6 fast path

Image or image+text:

  Qwen parser -> Kimi solver -> Llama verifier

Hard option-selection image tasks may run an extra focused Qwen option arbiter
pass when model answers disagree.


LOGS
----

Runtime logs:

     tail -f mcko.log

Useful patterns:

  - `Task solver API error`
  - `Qwen parser unavailable`
  - `Primary JSON parse failed`
  - `JSON repair pass`
  - `Accepting Qwen challenger answer`
  - `No safe final answer after consensus`


TESTING
-------

Local checks:

     python3 -m py_compile main.py app/*.py ui/*.py tests/*.py
     python3 -m unittest discover -s tests -p 'test_*.py'

Live regression suite:

     python3 tests/run_solver_fixture_suite.py user-regression

The live suite requires network and a valid OpenRouter key. It creates
screenshot-like images and sends image-only blocks through the same
`GeometryPhotoSolver` code path.


IMPORTANT NOTES
---------------

The app must solve arbitrary tasks, not just math or geometry. Do not add
hardcoded answers for screenshots.

The target environment is Linux/RosaOS with X11. Always manually verify hotkeys,
small window behavior, screenshot insertion, clipboard image flow, and
answer-only output on the target machine after UI changes.
